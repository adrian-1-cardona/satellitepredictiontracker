# Satellite Prediction Tracker — Full-Stack Integration Instructions

**Purpose:** Enforce consistent, full-stack integration between the FastAPI backend and React frontend, ensuring all services communicate correctly through a unified API contract.

---

## Architecture Overview

### Backend (FastAPI)
- **Base URL:** `http://localhost:8000` (development)
- **API Prefix:** `/api/v1`
- **Database:** PostgreSQL via SQLAlchemy ORM
- **Services:** Auth, Locations, Passes, Alerts, Admin
- **Deployment:** Docker container with Uvicorn

### Frontend (React + Vite)
- **Entry Point:** `frontend/src/main.jsx`
- **API Client:** `frontend/src/api/client.js` (axios-based)
- **Auth:** JWT tokens with refresh mechanism
- **State Management:** React Context (Auth) + Local State
- **Deployment:** Static build served by backend or separate server

### API Contract
The backend defines the contract; the frontend must strictly adhere to it:
- All requests use `/api/v1` prefix
- Responses follow backend schema (Pydantic models)
- Authentication via `Authorization: Bearer <access_token>`
- CORS enabled for frontend origin

## Core Rules for Full-Stack Integration

### 1. **API Client Module is the Single Source of Truth**

All HTTP communication **must** go through `frontend/src/api/client.js`. No direct axios calls or fetch in components.

**✅ DO:**
```javascript
// In a component
import { fetchLocations, createLocation } from '../api/client.js';

const data = await fetchLocations();
await createLocation({ name: 'My Site', latitude: 40.7, longitude: -74.0 });
```

**❌ DON'T:**
```javascript
// Direct axios call (breaks contract and error handling)
const response = await axios.get('http://localhost:8000/api/v1/locations');
// Direct fetch (bypasses auth interceptor)
const data = await fetch('/api/v1/locations');
```

**Why:** The client module centralizes auth token management, error handling, and API versioning. Direct calls bypass these safeguards.

---

### 2. **API Functions Must Match Backend Endpoints**

Backend routers define endpoints; frontend client functions must wrap them with identical contracts.

**Backend Example (FastAPI):**
```python
@router.get("/locations/{location_id}", response_model=LocationOut)
def get_location(location_id: int, current_user: User = Depends(get_current_user)) -> Location:
    return owned_location(db, current_user.id, location_id)
```

**Frontend Client Must Provide:**
```javascript
export async function fetchLocation(locationId) {
  const response = await api.get(`/locations/${locationId}`);
  return response.data;
}
```

**Usage in Component:**
```javascript
const location = await fetchLocation(locationId);
```

**Rules:**
- Function name should be camelCase and descriptive (e.g., `fetchLocation`, `createAlert`, `deletePasses`)
- Parameters must map to endpoint requirements
- Return value should be `response.data` (let client handle error responses)
- Always include JSDoc comments with backend endpoint reference

---

### 3. **Maintain Auth Token Lifecycle**

The client module handles token persistence and refresh automatically. Components must never manually manipulate tokens.

**✅ DO:**
```javascript
import { useAuth } from '../auth/AuthContext.jsx';

export function LoginForm() {
  const { login } = useAuth();

  const handleSubmit = async (email, password) => {
    try {
      const auth = await login({ email, password });
      // Auth is persisted automatically
      navigate('/dashboard');
    } catch (error) {
      setError(getErrorMessage(error));
    }
  };
}
```

**❌ DON'T:**
```javascript
// Manually setting tokens (bypass secure storage)
localStorage.setItem('token', response.data.access_token);
// Direct refresh calls (should be automatic)
await api.post('/auth/refresh', { refresh_token });
```

**Token Flow:**
1. `login()` or `register()` → backend returns tokens → `persistAuth()` saves and emits event
2. All requests auto-inject token via request interceptor
3. On 401 → response interceptor auto-refreshes token and retries request
4. On refresh failure → clear auth and redirect to landing

---

### 4. **Error Handling Must Use Centralized Helper**

All API calls should use `getErrorMessage()` to extract and format backend errors consistently.

**✅ DO:**
```javascript
import { fetchLocations, getErrorMessage } from '../api/client.js';

export function LocationList() {
  const [error, setError] = useState(null);

  useEffect(() => {
    fetchLocations()
      .catch((err) => {
        setError(getErrorMessage(err, 'Failed to load locations'));
      });
  }, []);
}
```

**❌ DON'T:**
```javascript
// Accessing error details directly (inconsistent across backend)
const msg = error.response.data.detail || error.message;
// Unformatted error display
console.error(error);
```

**Why:** Backend returns errors as either:
- `{ "detail": "..." }` (FastAPI validation)
- `{ "error": "..." }` (custom)
- Array of validation errors

`getErrorMessage()` normalizes these.

---

### 5. **Component State Patterns**

Components handling async data must implement three states: loading, data, error.

**✅ DO:**
```javascript
export function PassList({ locationId }) {
  const [passes, setPasses] = useState([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  useEffect(() => {
    setLoading(true);
    fetchPasses({ locationId })
      .then(setPasses)
      .catch((err) => setError(getErrorMessage(err)))
      .finally(() => setLoading(false));
  }, [locationId]);

  if (loading) return <Loader />;
  if (error) return <ErrorMessage msg={error} />;
  return <PassGrid passes={passes} />;
}
```

**❌ DON'T:**
```javascript
// Silent failures
useEffect(async () => {
  const data = await fetchPasses({ locationId });
  setPasses(data); // crashes if error, no feedback
}, []);

// Mixed loading states
const [isLoading, setLoading] = useState(false);
const [fetching, setFetching] = useState(false);
```

### 6. **Parameter Naming: Python Snake_Case ↔ JavaScript camelCase**

Backend uses `snake_case` for parameters; frontend client must convert.

**Backend Endpoint:**
```python
@router.get("/passes", response_model=list[PassOut])
def list_passes(
    location_id: int | None = None,
    days_ahead: int = Query(12, ge=1, le=30),
    satellite_name: str | None = None,
    min_elevation: float | None = Query(None, ge=0, le=90),
    ...
)
```

**Frontend Client:**
```javascript
export async function fetchPasses({ locationId, daysAhead = 12, satelliteName, minElevation }) {
  const params = {
    location_id: locationId,      // convert to snake_case
    days_ahead: daysAhead,
    satellite_name: satelliteName,
    min_elevation: minElevation,
  };
  const response = await api.get('/passes', { params });
  return response.data;
}
```

**Usage in Component:**
```javascript
// Use camelCase in component
const passes = await fetchPasses({ locationId: 5, daysAhead: 15, minElevation: 10 });
```

**Rule:** Client functions accept `camelCase` parameters and convert them to `snake_case` for the API.

### 7. **Backend Schema Compliance**

Frontend must serialize/deserialize data matching backend Pydantic models.

**Backend Schema (Pydantic):**
```python
class LocationCreate(BaseModel):
    name: str
    latitude: float
    longitude: float

class LocationOut(LocationCreate):
    id: int
    user_id: int
    created_at: datetime
```

**Frontend Usage:**
```javascript
// Sending (component data → backend format)
const payload = {
  name: 'Observatory',
  latitude: 40.7,
  longitude: -74.0,
};
await createLocation(payload); // client converts as needed

// Receiving (backend response → component display)
const location = await fetchLocation(1);
console.log(location); // { id: 1, name: '...', latitude: ..., user_id: ..., created_at: '...' }
```

**Rules:**
- Dates from backend are ISO 8601 strings; parse with `new Date(string)` when needed
- All numeric IDs from backend should not be modified
- Never mutate backend response objects directly; use `setState()` or copy
- Document expected response shape in client function JSDoc

### 9. **API Function Checklist**

When adding a new backend endpoint, the frontend client **must** implement:

1. ✅ **Function with JSDoc** — document endpoint, parameters, returns
2. ✅ **Parameter conversion** — snake_case conversion if needed
3. ✅ **Error handling** — let client interceptor handle
4. ✅ **Type hints** — JSDoc for parameters and return
5. ✅ **Export** — add to module exports for component import

**Template:**
```javascript
/**
 * Fetch satellite pass details.
 * @backend GET /passes/{pass_id}
 * @param {number} passId - The pass ID
 * @returns {Promise<Object>} Pass details (PassOut schema)
 * @throws {Error} On 404 (pass not found) or 401 (auth required)
 */
export async function fetchPass(passId) {
  const response = await api.get(`/passes/${passId}`);
  return response.data;
}
```

### 11. **Breaking Changes Management**

When backend API changes, frontend must update immediately.

**Scenario:** Backend `/passes` endpoint adds `pass_quality` filter.

**Backend (Fastapi):**
```python
@router.get("/passes")
def list_passes(
    ...,
    pass_quality: str | None = None,  # NEW
    ...
):
```

**Frontend Client (must update):**
```javascript
export async function fetchPasses({ locationId, daysAhead = 12, passQuality }) {
  const params = {
    location_id: locationId,
    days_ahead: daysAhead,
    pass_quality: passQuality,  // NEW
  };
  const response = await api.get('/passes', { params });
  return response.data;
}
```

**Components using this function must also update to pass new parameter or set default.**

## Full-Stack Workflow Example

### Scenario: Create a New Alert

**1. Backend defines endpoint (already done):**
```python
# backend/app/routers/alerts.py
@router.post("", response_model=AlertOut, status_code=status.HTTP_201_CREATED)
def create_alert(payload: AlertCreate, ...):
```

**2. Frontend adds client function:**
```javascript
// frontend/src/api/client.js
export async function createAlert(payload) {
  const response = await api.post('/alerts', payload);
  return response.data;
}
```

**3. Component imports and uses:**
```javascript
// frontend/src/pages/Alerts.jsx
import { createAlert, getErrorMessage } from '../api/client.js';

export function AlertForm() {
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  const handleSubmit = async (formData) => {
    setLoading(true);
    setError(null);
    try {
      const alert = await createAlert(formData);
      onSuccess(alert); // refetch list, show toast
    } catch (err) {
      setError(getErrorMessage(err, 'Failed to create alert'));
    } finally {
      setLoading(false);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      {/* form fields */}
      {error && <ErrorMessage msg={error} />}
      <button disabled={loading}>{loading ? 'Creating...' : 'Create'}</button>
    </form>
  );
}
```

## Common Pitfalls & Fixes

| Pitfall | Why It's Bad | Fix |
|---------|-------------|-----|
| Direct axios calls in components | Bypasses auth interceptor, inconsistent error handling | Use `api/client.js` functions |
| Hardcoded API URL | Breaks in different environments | Use `VITE_API_BASE_URL` env var |
| Manual token management | Insecure, duplicates logic | Use `AuthContext` and client interceptors |
| Ignoring loading/error states | Poor UX, silent failures | Always implement 3-state pattern |
| Mismatched parameter names | 400 Bad Request from backend | Convert camelCase ↔ snake_case in client |
| Direct response object mutation | Causes stale state bugs | Use setState or immutable copies |
| Unawaited async calls | Race conditions, lost requests | Always `await` or `.catch()` |
| API calls in render | Infinite loops, memory leaks | Use `useEffect` with dependency array |

## Questions & Debugging

**Q: Backend returned 401 but my component didn't logout.**
A: Check that the response interceptor in `client.js` is triggering. It only refreshes if `refreshToken` exists. Verify token is saved via `persistAuth()`.

**Q: My new API function isn't being called.**
A: Ensure it's `export`-ed in `client.js` and imported in the component. Use React DevTools to check if the function is defined.

**Q: Components show stale data after API call.**
A: Don't mutate response object. Use `setState()`: `setPasses([...response.data])` or `setPasses(response.data)` (spread unnecessary if not spreading).

**Q: CORS error when frontend calls backend.**
A: Check `CORS_ORIGINS` in backend `.env` includes frontend origin. Verify `fetch` call includes credentials if needed.

**Q: Docker backend won't start.**
A: Check `requirements.txt` is up-to-date. Run `docker-compose up --build`. Check logs: `docker logs <container>`.
