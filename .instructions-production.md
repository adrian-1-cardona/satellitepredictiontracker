# Production Instructions

## Overview

This document contains comprehensive production deployment and operation guidelines for the Satellite Tracker application.

## Architecture

### Technology Stack
- **Frontend**: React 19 + Vite + Three.js (being migrated to TypeScript + Tailwind)
- **Backend**: FastAPI + PostgreSQL + Redis + Celery
- **Monitoring**: Prometheus + Grafana + Loki
- **Deployment**: Docker Compose on VPS

### Services & Ports

| Service | Port | Purpose |
|---------|------|---------|
| Frontend | 3000 | Web UI |
| API | 8000 | REST API |
| PostgreSQL | 5432 | Database |
| Redis | 6379 | Cache & Message Broker |
| Prometheus | 9090 | Metrics collection |
| Grafana | 3001 | Dashboards & alerts |
| Loki | 3100 | Logs aggregation |
| PgAdmin | 5050 | Database admin (dev only) |

## Pre-Deployment Checklist

- [ ] All tests passing (backend: 33 tests, frontend: 46+ tests)

## Deployment

### Quick Start (5 minutes)

```bash
# 1. SSH to VPS
ssh admin@your-vps-address

# 2. Create app directory
mkdir -p /opt/satellite-tracker && cd /opt/satellite-tracker

# 3. Get docker-compose file
wget https://raw.githubusercontent.com/adrian-1-cardona/satellitepredictiontracker/main/docker-compose.prod.yml

# 4. Configure environment
wget https://raw.githubusercontent.com/adrian-1-cardona/satellitepredictiontracker/main/.env.example -O .env
nano .env  # Edit with production secrets

# 5. Deploy
docker-compose -f docker-compose.prod.yml pull
docker-compose -f docker-compose.prod.yml up -d

# 6. Wait and verify
sleep 30
curl http://localhost:8000/health
```

## Configuration

### Essential Environment Variables

| Variable | Purpose | Example |
|----------|---------|---------|
| `APP_PRODUCTION` | Production mode flag | `true` |
| `APP_DEBUG` | Debug mode (disable in production) | `false` |
| `SECRET_KEY` | JWT secret (32+ chars) | `your-random-secret-key` |
| `ADMIN_TOKEN` | Admin API docs access | `your-admin-token` |
| `DATABASE_URL` | PostgreSQL connection | `postgresql+psycopg2://tracker:password@postgres:5432/satellite_tracker` |
| `DB_PASSWORD` | Database password | `strong-password` |
| `ALLOWED_ORIGINS` | CORS origins | `https://yourdomain.com` |

### Performance Tuning

```bash
# Database pool sizing
DB_POOL_SIZE=20         # Increase for high concurrency
DB_MAX_OVERFLOW=30      # Overflow connections

# Rate limiting
RATE_LIMIT_REQUESTS=100  # Requests per window
RATE_LIMIT_WINDOW=60     # Time window in seconds

# Celery concurrency
# In docker-compose.prod.yml, celery_worker command:
celery -A app.tasks:celery_app worker --concurrency=4
```

## Security

### Security Headers

All responses include security headers:
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `X-XSS-Protection: 1; mode=block`
- `Referrer-Policy: strict-origin-when-cross-origin`

### API Documentation

Public documentation endpoints are **disabled**:
- ❌ `/docs` - Returns 404
- ❌ `/redoc` - Returns 404
- ❌ `/openapi.json` - Returns 404

Admin-only endpoints (require `ADMIN_TOKEN`):
- ✅ `/api/v1/admin/docs?token=YOUR_ADMIN_TOKEN`
- ✅ `/api/v1/admin/redoc?token=YOUR_ADMIN_TOKEN`
- ✅ `/api/v1/admin/openapi.json?token=YOUR_ADMIN_TOKEN`

### Database Security

1. **Change default credentials** in `.env`
2. **Backup encryption** (S3 with server-side encryption)
3. **Access control**: Database port (5432) should only be accessible internally
4. **Regular backups** with retention policy

## Monitoring & Alerting

### Key Metrics

**API Performance**:
- Request latency (p95 < 1 second)
- Error rate (< 1%)
- Request volume

**Database**:
- Connection pool utilization
- Slow query detection
- Backup status

**System**:
- CPU usage (< 80%)
- Memory usage (< 85%)
- Disk space (> 20% free)

### Viewing Metrics

1. **Prometheus**: http://your-vps:9090
2. **Grafana**: http://your-vps:3001
3. **CLI**: `docker exec satellite-tracker-api curl localhost:8000/metrics`

### Setting Up Alerts

In Grafana:
1. Create Alert Rules for key metrics
2. Configure notification channels (email, Slack, PagerDuty)
3. Set thresholds based on service requirements

## Troubleshooting

### Common Issues

**Issue**: API not responding
```bash
docker logs satellite-tracker-api
docker-compose -f docker-compose.prod.yml restart api
```

**Issue**: Database connection errors
```bash
docker exec satellite-tracker-postgres pg_isready
docker-compose -f docker-compose.prod.yml logs postgres
```

**Issue**: High memory usage
```bash
docker stats

```

## Upgrading

### Updating to New Version

1. **Pull latest code**
   ```bash
   git pull origin main
   ```

2. **Update docker-compose file**
   ```bash
   wget https://raw.githubusercontent.com/adrian-1-cardona/satellitepredictiontracker/main/docker-compose.prod.yml -O docker-compose.prod.yml
   ```

3. **Backup database**
   ```bash
   docker exec satellite-tracker-postgres pg_dump -U tracker satellite_tracker > backups/pre-upgrade.sql
   ```

4. **Pull and deploy**
   ```bash
   docker-compose -f docker-compose.prod.yml pull
   docker-compose -f docker-compose.prod.yml up -d
   ```

5. **Verify**
   ```bash
   curl http://localhost:8000/health
   docker logs satellite-tracker-api
   ```

| Monitoring | Optional | Required |
